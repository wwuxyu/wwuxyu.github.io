static

static修饰变量 两个用法 我们知道在函数内部定义的变量，当程序执行到它的定义处时，编译器为它在栈上分配空间，函数在栈上分配的空间在此函数执行结束时会释放掉，这样就产生了一个问题: 如果想将函数中此变量的值保存至下一次调用时，如何实现？ 最容易想到的方法是定义为全局的变量，但定义一个全局变量有许多缺点，最明显的缺点是破坏了此变量的访问范围（使得在此函数中定义的变量，不仅仅只受此函数控制）。static 关键字则可以很好的解决这个问题。

另外，在 C++ 中，需要一个数据对象为整个类而非某个对象服务,同时又力求不破坏类的封装性,即要求此成员隐藏在类的内部，对外不可见时，可将其定义为静态数据。

使用static修饰函数内的局部变量可以延长其声明周期并且不破坏其访问权限范围，下次调用该函数时值还是上次的。

同时，静态变量会自动初始化为0，但是我在函数内会声明多次怎么说？

实例代码

int increase() {
    static int cnt;
    cnt ++;
    std::cout << "current cnt " << cnt << std::endl;
}

int main() {
    increase();
    increase();
}
// current cnt 1
// current cnt 2

/*
static int cnt = 11;
current cnt 12
current cnt 13
*/

由于可以声明多次，所以我也可以多次赋值，但只有第一次赋值成功。

auto

在老版的c中是用来指定存储类型的，c++11中用来类型推导

  1. 声明变量时根据初始化表达式自动推断该变量的类型
  2. 声明函数时函数返回值的占位符
auto a = new auto(9); // int*

用法注意：

  1. 可以用valatile，pointer（*），reference（&），rvalue reference（&&） 来修饰auto

  2. 用auto声明的变量必须初始化(否则无法推导类型哦)

  3. auto不能与其他类型组合连用

  4. 函数参数和模板参数不能被声明为auto

  5. 定义在堆上的变量，使用了auto的表达式必须被初始化

  6. 以为auto是一个占位符，并不是一个他自己的类型，因此不能用于类型转换或其他一些操作，如sizeof和typeid

  7. auto不能自动推导成CV-qualifiers (constant & volatile qualifiers)

  8. auto会退化成指向数组的指针，除非被声明为引用
